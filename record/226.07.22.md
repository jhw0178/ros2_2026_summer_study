### 1. ROS 통신
#### ROS는 DDS를 이용해 통신하기 때문에 다음 사항을 확인해야 한다
    1. 서로 같은 네트워크에 있는가?
    2. 서로 같은 ROS_DOMAIN_ID를 사용하는가?
    3. 방화벽이 DDS 통신을 막고 있지는 않은가?
    4. WSL과 Windows가 서로 통신할 수 있는가?

##### 1-1. WSL 네트워크 설정 
##### wsl과 windows가 통신하려면 서로 같은 네트워크를 공유해야 한다
    - wsl -> window (mirror - network) : netshell
    - networkingMode=mirrored가 되면 네트워크가 같아져 통신이 가능함
##### 1-2. WSL 방화벽
##### Ubuntu에는 ufw라는 방화벽이 존재
    - $sudo ufw enable : DDS가 사용하는 UDP 포트가 막힐 수 있음
    - $sudo ufw disable : WSL 안에서 ROS 메시지가 막히지 않도록 설정
##### 1-3. WINDOWS 방화벽
##### Windows에는 Defender이라는 방화벽이 존재하며, ROS는 'UDP 브로드캐스트, UDP 멀티캐스트'를 많이 사용
    - Inbound Rule, Outbound Rule에서 허용
    - 방화벽을 아예 꺼도 됨
##### 1-4. ROS_DOMAIN_ID
##### 최종적으로 ROS_DOMAIN_ID가 다르면 통신이 불가능
    $export ROS_DOMAIN_ID=0
    위와 같은 값을 입력하여 도메인 값을 맞추면 됨

### 2. QoS
#### QoS 상태를 확인하고 해당 값의 의미를 이해한다
##### 2-1. turtlesim의 QoS
##### turtlesim을 실행하고 $ros2 param dump turtlesim 입력
    durability : volatile
    history : keep_last
    reliability : reliable
##### 2-2. History
##### 데이터 보관 크기 결정
    - KEEP_ALL : 모든 데이터를 보관
    - KEEP_LAST : Depth 값만큼만 보관
##### 2-3. Reliability
##### 데이터 처리 설정
    - BEST_EFFORT : 속도를 중시
    - RELIABLE : 데이터 수신에 집중
##### 2-4. Durability
##### 데이터 보관 유무
    - TRANSIENT_LOCAL : Subscription 생성 이전의 데이터부터 보관
    - VOLATILE : Subscription 생성 이전 데이터 무효
##### 2-5. Deadline
    정해진 주기 안에 데이터가 수·발신 되지 않을 경우 EventCallback을 실행
##### 2-6. Lifespan
    정해진 주기 안에 수신된 데이터만 유효 판정
##### 2-7. Liveliness
    정해진 주기 안에 노드 혹은 토픽의 생사를 확인
##### 2-8. rmw_qos_profile
    ROS2에서 QoS(Quality of Service) 설정을 담고 있는 구조체

### 3. QoS 반영한 Publisher, Subscriber
#### QoS 설정을 반영하여 노드를 생성해본다
##### 3-1. QoS 설정을 위한 library
    from rclpy.qos import QoSProfile
    from rclpy.qos import QoSHistoryPolicy
    from rclpy.qos import QoSReliabilityPolicy
    from rclpy.qos import QoSDurabilityPolicy
##### 3-2. QoS 값 설정
    self.qos_profile = QoSProfile(
            history=QoSHistoryPolicy.KEEP_ALL,  
            reliability=QoSReliabilityPolicy.RELIABLE, 
            durability=QoSDurabilityPolicy.TRANSIENT_LOCAL
        )
##### 3-3. QoS 반영한 Publisher
    self.qos_pub = self.create_publisher(String, "qos_test_topic", self.qos_profile)
##### 3-4. QoS 반영한 Subscriber
    self.qos_sub = self.create_subscription(String, "qos_test_topic", self.qos_sub_callback, 10)

### 4. 표준 단위, 좌표 표현
#### ROS에서는 국제단위계인 SI unit을 사용한다
##### 4-1. 표준 단위
    - 길이 : 미터 [m]
    - 무게 : 키로그램 [kg]
    - 시간 : 초 [s]
    - 전류 : 암페어 [A]
    - 평면각 : 라디안 [rad]
    - 주파수 : 헤르츠 [Hz]
    - 온도 : 셀시우스 [Celsius]
    - 전압 : 볼트 [V]
##### 4-2. 좌표 표현
##### 오른손 법칙을 따른다
    - 반시계 방향 : +
    - 좌표계 : ENU 
        E : East (Red)
        N : North (Green)
        U : Up (Blue)
    - 표현 방식이 다르면 접미사로 _optical, _ned를 붙여서 구분
    - 회전의 표현 방식 : 쿼터니언 x,y,z,w
